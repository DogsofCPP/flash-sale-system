# 消息队列对比

## 1. 主流消息队列对比

| 特性 | RabbitMQ | Kafka | RocketMQ |
|------|----------|-------|----------|
| 吞吐量 | 中等(万级) | 高(十万级) | 高(十万级) |
| 延迟 | 低(微秒级) | 毫秒级 | 毫秒级 |
| 事务支持 | 弱 | 强 | 强 |
| 顺序消息 | 支持 | 支持 | 支持 |
| 延迟消息 | 支持(插件) | 不支持 | 原生支持 |
| 消息回溯 | 不支持 | 支持 | 支持 |
| 生态 | 成熟 | 非常成熟 | 阿里生态 |
| 学习曲线 | 低 | 中 | 中 |

## 2. 选型建议

### 2.1 RabbitMQ适用场景

- 业务系统间的异步通信
- 复杂的消息路由场景
- 需要可靠消息传输
- 小到中等规模系统

### 2.2 Kafka适用场景

- 日志收集、实时分析
- 高吞吐量的大数据场景
- 秒杀、直播等高并发场景
- 需要消息回溯的场景

### 2.3 RocketMQ适用场景

- 电商订单系统
- 需要事务消息
- 延迟消息场景
- 阿里云环境

## 3. 本系统选择

本系统选择**RabbitMQ**，原因：

1. 配置简单，学习成本低
2. 功能满足秒杀场景需求
3. Spring Boot集成完善
4. 管理界面友好
