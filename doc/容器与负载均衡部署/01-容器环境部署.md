# 容器环境部署

## 1. 环境准备

### 1.1 安装Docker

**Windows:**
1. 下载 Docker Desktop: https://www.docker.com/products/docker-desktop
2. 运行安装程序
3. 启动Docker Desktop

**Linux (Ubuntu):**
```bash
curl -fsSL https://get.docker.com | sh
sudo systemctl start docker
sudo systemctl enable docker
```

### 1.2 安装Docker Compose

```bash
# Windows/Mac: Docker Desktop已包含
# Linux:
sudo curl -L "https://github.com/docker/compose/releases/download/v2.20.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose
```

### 1.3 验证安装

```bash
docker --version
docker-compose --version
docker info
```

## 2. 配置镜像加速（国内用户）

创建或编辑 `~/.docker/daemon.json`:

```json
{
    "registry-mirrors": [
        "https://docker.m.daocloud.io",
        "https://dockerproxy.cn",
        "https://mirror.ccs.tencentyun.com"
    ]
}
```

重启Docker Desktop使配置生效。

## 3. 拉取基础镜像

```bash
# 拉取MySQL
docker pull mysql:8.0

# 拉取Redis
docker pull redis:7-alpine

# 拉取RabbitMQ
docker pull rabbitmq:3-management

# 拉取Nginx
docker pull nginx:alpine
```

## 4. 构建应用镜像

```bash
cd g:\sale_system\flash-sale-system

# 构建用户服务镜像
docker build -t user-service:latest -f user-service/Dockerfile .

# 或使用docker-compose自动构建
docker-compose build
```

## 5. 启动服务

```bash
# 后台启动所有服务
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f
```

## 6. 访问服务

| 服务 | 地址 |
|------|------|
| 前端页面 | http://localhost |
| API接口 | http://localhost:8080 |
| RabbitMQ管理 | http://localhost:15672 |

## 7. 清理环境

```bash
# 停止服务
docker-compose down

# 停止并删除数据卷
docker-compose down -v

# 删除所有镜像
docker-compose down --rmi all
```

## 8. 常见问题

### 8.1 Docker Desktop无法启动

确保开启了Windows Hyper-V或WSL2支持。

### 8.2 端口冲突

修改docker-compose.yml中的端口映射。

### 8.3 内存不足

调整Docker Desktop设置中的内存分配（建议8GB+）。
