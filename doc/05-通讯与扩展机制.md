# 通讯与扩展机制

## 1. 服务间通信概述

微服务架构下，服务间通信分为**同步调用**和**异步消息**两类：

| 通信模式 | 技术方案 | 适用场景 | 特点 |
|---------|---------|---------|------|
| 同步 HTTP | Spring Cloud OpenFeign | 查询类、实时性要求高的场景 | 实时响应，调用方阻塞等待 |
| 异步消息 | RabbitMQ / Kafka | 订单创建、状态通知等解耦场景 | 削峰填谷，发布-订阅 |
| 服务注册发现 | Nacos | 所有服务间通信 | 服务名路由，负载均衡 |

## 2. RabbitMQ 消息队列

### 2.1 组件拓扑

```
┌─────────────┐         ┌──────────────────┐         ┌──────────────────┐
│  Producer   │────────→│     Exchange      │────────→│     Queue        │
│ (秒杀服务)   │         │  (Direct/Fanout)  │         │  (持久化)         │
└─────────────┘         └──────────────────┘         └────────┬─────────┘
                                                              │
                                          ┌───────────────────┼───────────────────┐
                                          │                   │                   │
                                          ▼                   ▼                   ▼
                               ┌──────────────────┐ ┌──────────────────┐ ┌──────────────────┐
                               │  Consumer        │ │  Consumer        │ │   Dead Letter    │
                               │ (订单服务)        │ │ (秒杀服务)        │ │   Queue (DLQ)    │
                               └──────────────────┘ └──────────────────┘ └──────────────────┘
```

### 2.2 交换机与队列配置

| 交换机名称 | 类型 | 绑定队列 | 路由键 | 用途 |
|-----------|------|---------|-------|------|
| flash.exchange | Direct | flash.seckill.order.queue | flash.seckill.order | 秒杀下单消息 |
| flash.exchange | Direct | flash.order.status.queue | flash.order.status | 订单状态变更通知 |
| flash.order.cancel.exchange | Direct | flash.stock.rollback.queue | flash.order.cancel | 订单取消触发库存回滚 |
| flash.dlx.exchange | Direct | flash.seckill.dlq | flash.seckill.dead | 死信队列 |

### 2.3 核心配置类

```java
@Configuration
public class RabbitMQConfig {

    public static final String EXCHANGE_NAME = "flash.exchange";
    public static final String ORDER_QUEUE = "flash.seckill.order.queue";
    public static final String ORDER_ROUTING_KEY = "flash.seckill.order";
    public static final String DLX_EXCHANGE = "flash.dlx.exchange";
    public static final String DLQ_NAME = "flash.seckill.dlq";

    @Bean
    public DirectExchange flashExchange() {
        return new DirectExchange(EXCHANGE_NAME, true, false);
    }

    @Bean
    public Queue orderQueue() {
        return QueueBuilder.durable(ORDER_QUEUE)
            .withArgument("x-dead-letter-exchange", DLX_EXCHANGE)
            .withArgument("x-dead-letter-routing-key", "flash.seckill.dead")
            .build();
    }

    @Bean
    public Binding orderBinding() {
        return BindingBuilder.bind(orderQueue())
            .to(flashExchange())
            .with(ORDER_ROUTING_KEY);
    }

    @Bean
    public DirectExchange dlxExchange() {
        return new DirectExchange(DLX_EXCHANGE, true, false);
    }

    @Bean
    public Queue deadLetterQueue() {
        return QueueBuilder.durable(DLQ_NAME)
            .withArgument("x-message-ttl", 1800000)  // 30分钟
            .build();
    }
}
```

## 4. 消息可靠性保障

### 4.1 生产者可靠性

```java
@Component
public class ReliableMessageProducer {

    @Autowired
    private RabbitTemplate rabbitTemplate;

    @PostConstruct
    public void init() {
        // 消息发送确认回调
        rabbitTemplate.setConfirmCallback((correlationData, ack, cause) -> {
            if (!ack) {
                log.error("消息发送失败，ID: {}, 原因: {}", correlationData, cause);
                // 可以写入本地日志表，后续补偿重发
                messageRetryService.saveFailedMessage(correlationData);
            }
        });

        // 消息路由失败回调
        rabbitTemplate.setReturnsCallback(returned -> {
            log.error("消息路由失败: exchange={}, routingKey={}, replyCode={}",
                returned.getExchange(), returned.getRoutingKey(), returned.getReplyCode());
        });
    }

    public void sendOrderMessage(SeckillOrderMessage message) {
        CorrelationData correlationData = new CorrelationData(message.getOrderNo());
        rabbitTemplate.convertAndSend(
            RabbitMQConfig.EXCHANGE_NAME,
            RabbitMQConfig.ORDER_ROUTING_KEY,
            message,
            correlationData
        );
    }
}
```

### 4.2 消费者可靠性

```java
@Service
public class OrderMessageConsumer {

    @RabbitListener(queues = RabbitMQConfig.ORDER_QUEUE, concurrency = "3-10")
    public void consumeSeckillOrder(SeckillOrderMessage message,
                                   Channel channel,
                                   @Header(AmqpHeaders.DELIVERY_TAG) long tag) {
        String orderNo = message.getOrderNo();

        try {
            // 1. 幂等性检查
            if (orderService.existsByOrderNo(orderNo)) {
                channel.basicAck(tag, false);
                return;
            }

            // 2. 创建订单
            orderService.createOrder(message);

            // 3. 确认消息
            channel.basicAck(tag, false);

        } catch (DuplicateKeyException e) {
            // 唯一键冲突，说明已处理，直接确认
            channel.basicAck(tag, false);
        } catch (Exception e) {
            log.error("消费消息失败: {}", orderNo, e);
            // 消费失败，NACK 并重新入队（最多重试3次）
            channel.basicNack(tag, false, true);
        }
    }
}
```

## 5. 异步事件驱动

### 5.1 Spring ApplicationEvent

系统内部使用 Spring 事件机制实现业务解耦：

```java
// 定义秒杀成功事件
public class SeckillSuccessEvent extends ApplicationEvent {
    private final Long activityId;
    private final Long userId;
    private final Long productId;
    private final String orderNo;

    public SeckillSuccessEvent(Object source, Long activityId, Long userId,
                               Long productId, String orderNo) {
        super(source);
        this.activityId = activityId;
        this.userId = userId;
        this.productId = productId;
        this.orderNo = orderNo;
    }
    // getters...
}

// 发布事件
@Service
public class SeckillServiceImpl {

    @Autowired
    private ApplicationEventPublisher eventPublisher;

    public void processSeckill(SeckillRequest request) {
        // 秒杀核心逻辑...
        SeckillSuccessEvent event = new SeckillSuccessEvent(
            this, request.getActivityId(), request.getUserId(),
            request.getProductId(), orderNo
        );
        eventPublisher.publishEvent(event);
    }
}

// 监听事件
@Service
public class SeckillEventListener {

    @Autowired
    private WebSocketService webSocketService;

    @EventListener
    public void handleSuccess(SeckillSuccessEvent event) {
        webSocketService.notifyUser(event.getUserId(),
            "秒杀成功，订单号: " + event.getOrderNo());
    }
}
```

### 5.2 事件类型定义

| 事件类 | 触发时机 | 监听器处理 |
|-------|---------|---------|
| SeckillSuccessEvent | 秒杀成功，MQ消息发送后 | WebSocket 通知用户 |
| SeckillFailedEvent | 秒杀失败（库存不足/限购） | 日志记录 |
| OrderCreatedEvent | 订单创建成功 | 更新秒杀订单状态 |
| OrderCancelledEvent | 订单取消/超时 | 触发库存回滚 |
| StockWarmedUpEvent | 库存预热完成 | 更新活动状态为"进行中" |

## 6. 扩展机制

### 6.1 SPI 插件化设计

系统支持通过 Java SPI 机制扩展功能，便于在不修改核心代码的情况下增加新能力：

```java
// 插件接口定义
public interface SeckillInterceptor {

    /**
     * 秒杀前置拦截
     * @return true 继续执行，false 阻止秒杀
     */
    boolean beforeSeckill(SeckillContext context);

    /**
     * 秒杀后置处理
     */
    void afterSeckill(SeckillContext context, SeckillResult result);
}

// 插件实现（编译到 classpath 中，通过 SPI 加载）
public class BlacklistInterceptor implements SeckillInterceptor {

    @Override
    public boolean beforeSeckill(SeckillContext context) {
        // 黑名单用户直接拦截
        return !blacklistService.isBlackUser(context.getUserId());
    }

    @Override
    public void afterSeckill(SeckillContext context, SeckillResult result) {
        // 黑名单用户操作记录
    }
}

// SPI 加载
@Service
public class InterceptorChain {

    private final List<SeckillInterceptor> interceptors;

    public InterceptorChain() {
        ServiceLoader<SeckillInterceptor> loader =
            ServiceLoader.load(SeckillInterceptor.class);
        this.interceptors = new ArrayList<>();
        loader.forEach(interceptors::add);
    }

    public boolean doBefore(SeckillContext context) {
        for (SeckillInterceptor interceptor : interceptors) {
            if (!interceptor.beforeSeckill(context)) {
                return false;
            }
        }
        return true;
    }
}
```

## 8. 重试策略

| 失败场景 | 重试策略 | 最大次数 | 退避方式 |
|---------|---------|---------|---------|
| MQ 消息发送失败 | 本地消息表 + 定时任务重发 | 3次 | 指数退避（1s, 2s, 4s） |
| 订单创建失败 | MQ 重试机制 | 无限重试直到成功 | 指数退避 |
| 库存同步失败 | 补偿队列延迟重试 | 5次 | 固定5秒 |
| Nacos 配置拉取失败 | 快速失败 + 本地缓存兜底 | - | - |
| OpenFeign 调用超时 | Feign 重试（仅 GET） | 2次 | Ribbon 默认 |

## 9. 消息追踪

为每个消息生成唯一追踪 ID，通过 MDC 打印到日志：

```java
// 消息发送时添加追踪ID
public void sendMessage(SeckillOrderMessage message) {
    String traceId = UUID.randomUUID().toString().replace("-", "");
    MDC.put("traceId", traceId);
    message.setTraceId(traceId);

    log.info("发送秒杀订单消息: {}", JSON.toJSONString(message));
    rabbitTemplate.convertAndSend(...);

    MDC.remove("traceId");
}

// 消费者接收时读取追踪ID
@RabbitListener(...)
public void handleMessage(SeckillOrderMessage message) {
    if (message.getTraceId() != null) {
        MDC.put("traceId", message.getTraceId());
    }
    try {
        // 业务处理
        log.info("消费秒杀订单消息: orderNo={}", message.getOrderNo());
    } finally {
        MDC.remove("traceId");
    }
}
```
