# 数据库设计

## 1. 数据库架构

系统采用**多数据库分域部署**策略，每个业务域拥有独立的 MySQL 数据库实例，独立维护，互不干扰，便于按域扩容和故障隔离。

| 数据库名称 | 端口 | 用途 | 归属服务 |
|-----------|------|------|---------|
| flash_sale_user | 3306 | 用户认证数据 | user-service |
| flash_sale_product | 3306 | 商品信息数据 | product-service |
| flash_sale_order | 3306 | 订单流转数据 | order-service |
| flash_sale | 3306 | 秒杀活动与库存数据 | seckill-service |
| nacos_config | 3306 | Nacos 配置持久化 | Nacos |

初始化脚本统一放置在 `docker/init.sql`，由 MySQL Master 容器启动时自动执行，确保数据库结构与数据在首次部署时完整就绪。

## 2. 表结构设计

### 2.1 用户数据表（flash_sale_user）

**用户基础表 `t_user`**

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 主键 |
| username | VARCHAR(64) | NOT NULL, UNIQUE | 用户名 |
| password_hash | VARCHAR(255) | NOT NULL | BCrypt 加密密码 |
| phone | VARCHAR(32) | UNIQUE | 手机号 |
| status | TINYINT | NOT NULL, DEFAULT 1 | 状态：1-正常，0-禁用 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

```sql
CREATE TABLE t_user (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(64) NOT NULL UNIQUE COMMENT '用户名',
    password_hash VARCHAR(255) NOT NULL COMMENT 'BCrypt加密密码',
    phone VARCHAR(32) UNIQUE COMMENT '手机号',
    status TINYINT NOT NULL DEFAULT 1 COMMENT '1-正常 0-禁用',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_username (username),
    INDEX idx_phone (phone)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 2.2 商品数据表（flash_sale_product）

**商品表 `t_product`**

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 主键 |
| name | VARCHAR(128) | NOT NULL | 商品名称 |
| description | VARCHAR(512) | | 商品描述 |
| price | DECIMAL(10,2) | NOT NULL | 原价 |

| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

```sql
CREATE TABLE t_product (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,

    description VARCHAR(512) COMMENT '商品描述',
```

### 2.3 订单数据表（flash_sale_order）

**订单主表 `t_order`**

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 主键 |
| order_no | VARCHAR(64) | NOT NULL, UNIQUE | 订单编号（雪花算法） |
| user_id | BIGINT | NOT NULL | 用户ID |
| product_id | BIGINT | NOT NULL | 商品ID |
| activity_id | BIGINT | | 关联秒杀活动ID |
| product_name | VARCHAR(128) | NOT NULL | 下单时商品名称（冗余） |
| product_price | DECIMAL(10,2) | NOT NULL | 下单时单价（冗余） |
| quantity | INT | NOT NULL, DEFAULT 1 | 购买数量 |
| total_amount | DECIMAL(10,2) | NOT NULL | 订单总金额 |
| status | TINYINT | NOT NULL, DEFAULT 0 | 订单状态 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |
| pay_time | DATETIME | | 支付时间 |
| expire_time | DATETIME | | 订单过期时间 |
| cancel_time | DATETIME | | 取消时间 |

> **冗余设计**：`product_name` 和 `product_price` 在下单时冗余存储，避免后续商品价格变动影响历史订单的准确性。

```sql
CREATE TABLE t_order (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    order_no VARCHAR(64) NOT NULL UNIQUE COMMENT '订单编号',
    user_id BIGINT NOT NULL COMMENT '用户ID',
    product_id BIGINT NOT NULL COMMENT '商品ID',
    activity_id BIGINT COMMENT '秒杀活动ID',
    product_name VARCHAR(128) NOT NULL COMMENT '商品名称',
    product_price DECIMAL(10,2) NOT NULL COMMENT '购买单价',
    quantity INT NOT NULL DEFAULT 1 COMMENT '购买数量',
    total_amount DECIMAL(10,2) NOT NULL COMMENT '订单总金额',
    status TINYINT NOT NULL DEFAULT 0 COMMENT '0-待支付 1-已支付 2-已取消 3-已退款',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    pay_time DATETIME COMMENT '支付时间',
    expire_time DATETIME COMMENT '订单过期时间',
    cancel_time DATETIME COMMENT '取消时间',
    INDEX idx_user (user_id),
    INDEX idx_order_no (order_no),
    INDEX idx_status (status),
    INDEX idx_expire (expire_time),
    INDEX idx_created (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

**订单状态定义**

| 状态码 | 状态名 | 说明 |
|--------|--------|------|
| 0 | 待支付 | 订单已创建，等待用户付款 |
| 1 | 已支付 | 用户完成付款 |
| 2 | 已取消 | 用户主动取消 |
| 3 | 已退款 | 退款完成 |

| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

**秒杀商品表 `seckill_product`**

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 主键 |
| activity_id | BIGINT | NOT NULL | 关联活动ID |
| product_id | BIGINT | NOT NULL | 关联商品ID |
| seckill_price | DECIMAL(10,2) | NOT NULL | 秒杀价格 |
| stock | INT | NOT NULL, DEFAULT 0 | 秒杀库存 |
| sold_count | INT | NOT NULL, DEFAULT 0 | 已售数量 |
| limit_per_user | INT | NOT NULL, DEFAULT 1 | 每人限购数量 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

> **sold_count 与 stock 的关系**：`stock` 是初始总库存，`sold_count` 是累计已售数量，两者之差（stock - sold_count）为当前可用库存。`seckill_product` 表中的库存用于 MySQL 层兜底扣减。

```sql
CREATE TABLE seckill_product (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    activity_id BIGINT NOT NULL COMMENT '秒杀活动ID',
    product_id BIGINT NOT NULL COMMENT '商品ID',
    seckill_price DECIMAL(10,2) NOT NULL COMMENT '秒杀价',
    stock INT NOT NULL DEFAULT 0 COMMENT '秒杀库存',
    sold_count INT NOT NULL DEFAULT 0 COMMENT '已售数量',
    limit_per_user INT NOT NULL DEFAULT 1 COMMENT '每人限购数量',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_activity (activity_id),
    INDEX idx_product (product_id),
    UNIQUE KEY uk_activity_product (activity_id, product_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|

```sql
    status TINYINT NOT NULL DEFAULT 0 COMMENT '0-排队中 1-成功 2-失败 3-超时',

    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_user_activity (user_id, activity_id),
    INDEX idx_order_no (order_no),
    INDEX idx_status (status),
    UNIQUE KEY uk_user_activity_product (user_id, activity_id, product_id)
```


## 3. Redis 数据结构

系统在 Redis 中使用 String、Hash 两种数据结构，分别对应不同的缓存场景：

### 3.1 秒杀库存计数器

```
Key:   flash:seckill:stock:{activityId}:{productId}
Type:  String
Value: 当前可用库存（整数）
TTL:   活动结束时间 + 1小时
```

### 3.2 用户购买记录

```
Key:   flash:seckill:user:{activityId}:{productId}:{userId}
Type:  String
Value: 用户已购买数量
TTL:   活动结束时间 + 1小时
```

### 3.3 活动详情缓存

```
Key:   flash:activity:info:{activityId}
Type:  Hash
Field: name / start_time / end_time / status / seckill_price
Value: 对应值
TTL:   5分钟
```

## 4. MySQL 主从复制配置

### 4.1 Master 配置（3307）

```ini
[mysqld]
server-id = 1
log-bin = mysql-bin
binlog-format = ROW
gtid-mode = ON
enforce-gtid-consistency = ON
sync-binlog = 1
innodb_flush_log_at_trx_commit = 1
```

### 4.2 Slave 配置（3308）

```ini
[mysqld]
server-id = 2
relay-log = relay-bin
read-only = 1
gtid-mode = ON
replica-skip-errors = 1062
```

## 5. 索引策略汇总

| 表名 | 索引字段 | 类型 | 用途 |
|------|---------|------|------|
| t_user | username | 普通索引 | 按用户名快速查询 |
| t_user | phone | 普通索引 | 按手机号快速查询 |
| t_order | user_id | 普通索引 | 用户订单查询 |
| t_order | (user_id, status) | 复合索引 | 用户按状态查订单 |
| t_order | expire_time, status | 复合索引 | 扫描超时订单 |
| seckill_activity | (start_time, end_time) | 复合索引 | 按时间段查询活动 |
| seckill_activity | status | 普通索引 | 按状态筛选活动 |
| seckill_product | activity_id | 普通索引 | 查询活动下商品 |
| seckill_product | (activity_id, product_id) | 唯一索引 | 防重复绑定 |
| seckill_order | (user_id, activity_id) | 复合索引 | 用户活动维度查询 |
| seckill_order | (user_id, activity_id, product_id) | 唯一索引 | 数据库层一人一单保障 |
| stock_flow | (activity_id, product_id) | 复合索引 | 按活动商品查流水 |
