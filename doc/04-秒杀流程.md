# 核心秒杀流程详解

## 1. 秒杀场景特点

| 特点 | 描述 | 应对策略 |
|------|------|----------|
| 瞬时高并发 | 活动开始瞬间涌入大量请求 | 限流 + 异步队列 |
| 库存有限 | 热门商品数量少 | Redis预减库存 |
| 防刷防爬 | 恶意请求占用资源 | 验证码 + 限流 |
| 库存超卖 | 库存为负数 | Lua脚本原子操作 |

## 3. 核心代码

### 3.1 秒杀接口

```java
@PostMapping("/seckill")
public Result<SeckillResult> seckill(SeckillRequest request) {
    // 1. 参数校验
    ValidationUtils.validate(request);

    // 2. 限流检查
    if (!rateLimiter.tryAcquire()) {
        return Result.rateLimit();
    }

    // 3. 热点数据预热检查
    String stockKey = "seckill:stock:" + request.getProductId();
    Long stock = redisTemplate.opsForValue().decrement(stockKey);

    // 4. 库存不足直接返回
    if (stock < 0) {
        redisTemplate.opsForValue().increment(stockKey);
        return Result.stockNotEnough();
    }

    // 5. 发送异步消息创建订单
    OrderMessage message = new OrderMessage(...);
    kafkaTemplate.send("seckill-order-topic", message);

    // 6. 立即返回排队中
    return Result.success(new SeckillResult("QUEUED", request.getOrderNo()));
}
```

### 3.2 Lua脚本防超卖

```lua
-- stock.lua
local stock_key = KEYS[1]
local user_key = KEYS[2]
local stock = redis.call('GET', stock_key)

if not stock then
    return -1  -- 库存未初始化
end

if tonumber(stock) <= 0 then
    return 0   -- 库存不足
end

-- 检查用户是否已购买
if redis.call('SISMEMBER', user_key, ARGV[1]) == 1 then
    return -2  -- 已购买
end

-- 扣减库存
redis.call('DECR', stock_key)
redis.call('SADD', user_key, ARGV[1])
return 1  -- 成功
```

### 3.3 订单异步创建

```java
@KafkaListener(topics = "seckill-order-topic")
public void createOrder(OrderMessage msg) {
    // 1. 幂等检查
    if (orderMapper.existsByOrderNo(msg.getOrderNo())) {
        return;
    }

    // 2. 创建订单
    Order order = new Order();
    order.setOrderNo(msg.getOrderNo());
    order.setUserId(msg.getUserId());
    order.setProductId(msg.getProductId());
    order.setQuantity(1);
    order.setStatus(OrderStatus.PENDING);

    orderMapper.insert(order);

    // 3. 更新秒杀记录
    seckillOrderMapper.insert(seckillOrder);

    // 4. 发送支付通知
    paymentService.notifyUser(order);
}
```

## 4. 关键指标

| 阶段 | 目标 | 实现 |
|------|------|------|
| 请求拦截 | 过滤99%无效请求 | 验证码 + 限流 |
| 库存判断 | <1ms | Redis原子操作 |
| 订单创建 | 异步不阻塞 | Kafka消息队列 |
| 最终一致性 | 订单不丢失 | 消息确认机制 |
