## 2. 限流方案

### 2.1 限流算法对比

| 算法 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 计数器 | 简单易懂 | 突发流量不均匀 | 固定QPS限制 |
| 滑动窗口 | 精确、均匀 | 实现复杂 | 精确限流 |
| 漏桶 | 流量平滑 | 不适合突发 | API限流 |
| 令牌桶 | 支持突发 | - | 秒杀场景 ✓ |

### 2.2 令牌桶实现

```java
@Component
public class RateLimiter {

    private final Map<String, RateLimiter> limiters = new ConcurrentHashMap<>();

    public boolean tryAcquire(String key, int permits, long duration, TimeUnit unit) {
        RateLimiter limiter = limiters.computeIfAbsent(key,
            k -> RateLimiter.create(permits / (double) duration));

        return limiter.tryAcquire(permits);
    }
}
```

### 2.3 分布式限流（Redis）

```java
public class RedisRateLimiter {

    public boolean isAllowed(String userId, String action) {
        String key = "rate:" + action + ":" + userId;
        long count = redisTemplate.opsForValue().increment(key);

        if (count == 1) {
            redisTemplate.expire(key, 1, TimeUnit.SECONDS);
        }

        return count <= MAX_REQUESTS_PER_SECOND;
    }
}
```

### 3.2 缓存问题处理

| 问题 | 现象 | 解决方案 |
|------|------|----------|
| 缓存穿透 | 查询不存在的数据 | 布隆过滤器 + 空值缓存 |
| 缓存击穿 | 热点key过期瞬间 | 互斥锁 + 永不过期 |
| 缓存雪崩 | 大量key同时过期 | 过期时间随机 + 预热 |

```java
// 布隆过滤器防止缓存穿透
@Service
public class BloomFilterService {

    private final BloomFilter<String> filter = BloomFilter.create(
        Funnels.stringFunnel(StandardCharsets.UTF_8),
        1000000,  // 预期插入数
        0.001     // 误判率
    );

    public boolean mightContain(String key) {
        return filter.mightContain(key);
    }

    public void add(String key) {
        filter.put(key);
    }
}
```

### 4.2 消息可靠性保证

```java
// 生产者端 - 消息确认
@Service
public class OrderMessageProducer {

    public void sendOrderMessage(OrderMessage message) {
        kafkaTemplate.send("order-topic", message.getOrderNo(), message)
            .addCallback(
                success -> log.info("消息发送成功: {}", message.getOrderNo()),
                failure -> {
                    log.error("消息发送失败: {}", message.getOrderNo());
                    // 发送到死信队列或重试
                    sendToDeadLetterQueue(message);
                }
            );
    }
}

// 消费者端 - 手动提交
@KafkaListener(topics = "order-topic")
public void consumeOrderMessage(OrderMessage message, Acknowledgment ack) {
    try {
        processOrder(message);
        ack.acknowledge();  // 手动确认
    } catch (Exception e) {
        log.error("处理订单失败: {}", message.getOrderNo(), e);
        // 不确认，让消息重试
        throw e;
    }
}
```

## 5. 分布式锁

### 5.1 Redisson分布式锁

```java
@Service
public class DistributedLockService {

    @Autowired
    private RedissonClient redissonClient;

    public <T> T executeWithLock(String lockKey, long waitTime, TimeUnit unit,
                                   Supplier<T> action) {
        RLock lock = redissonClient.getLock(lockKey);
        boolean acquired = lock.tryLock(waitTime, unit);
        if (!acquired) {
            throw new BusinessException("系统繁忙，请稍后重试");
        }
        try {
            return action.get();
        } finally {
            if (lock.isHeldByCurrentThread()) {
                lock.unlock();
            }
        }
    }
}
```
